Что сделано в этой итерации и зачем

Резюме

Цель итерации — перенести строгие математические и финансовые методы из набора инструментов ARPM (arpym) в нашу платформу кредитных рисков, не добавляя внешних зависимостей. Каждый метод реализован как самодостаточный порт на чистых numpy/scipy и встроен в существующие модули, чтобы заменить «заданные вручную» допущения оценками на основе данных.

📈 От допущений к оценкам

Раньше: матрица корреляций бралась из графа «как есть», нагрузки факторов задавались равными, параметры копулы конфигурировались. Теперь все три можно оценить из наблюдаемых данных и сравнить с конфигурацией.

📦 Порты, а не зависимости

ARPM не становится зависимостью. statsmodels и skrmt заменены реализациями на numpy/scipy. Стек остаётся: numpy, pandas, scipy, scikit-learn, matplotlib.

🧩 Модульность для команды

ETL-слой делит работу на независимые этапы, которые общаются только через диск (ArtifactStore). Разные люди владеют разными этапами; каждый ноутбук запускается отдельно.

✅ Совместимость

Все существующие публичные API сохранены. Новые методы добавлены как дополнительные пути (с флагами denoise=, apply=), старые остаются поведением по умолчанию.

Главный результатРиск-команда получила инструменты, отвечающие на вопрос «откуда взялись параметры?»: матрицы миграции рейтингов — из когорт, корреляции — очищенные от шума, факторные нагрузки — подогнанные, параметры копулы — откалиброванные по дефолтам. Всё с проверкой против известных математических ответов.

  • Независимость → мера зависимости = 0.
  • Полная зависимость → мера = 1.

  • Подгонка факторов → восстановление известной нагрузки.
  • Матрица миграции → монотонная PD по рейтингу.

M1

Энтропийный пулинг — relative_entropy.py

Примитив

Базовый «строительный блок». Находит распределение вероятностей, минимально отличающееся от априорного (по дивергенции Кульбака—Лейблера), но удовлетворяющее заданным ограничениям на моменты. Это строгая основа для стресс-видов и для условных вероятностей (M4) и матриц миграции (M2).

p* = argmin KL(p ‖ p_prior) при условии: ∑ pᵢ = 1, E_p[g(x)] = b (целевые моменты)
ПроверкаНа синтетических данных восстанавливает заданное среднее и дисперсию с точностью до численной ошибки. Порт min_rel_entropy_sp из arpym.views; зависимость от statsmodels.DescrStatsW заменена на взвешенные среднее/дисперсию на numpy.

Источник ARPM: views.min_rel_entropy_sp · Тест: test_relative_entropy

M2

Матрицы миграции рейтингов — credit_transitions.py

Кредитный риск

Оценивает матрицу переходов между рейтингами из когортных данных (сколько заёмщиков перешло из рейтинга i в j). Использует непрерывно-временной генератор и матричную экспоненту — корректнее, чем простой подсчёт частот, потому что гарантирует валидную стохастическую матрицу и согласованность по времени.

gᵢⲤ = N_cum[i,j] / ∑ₜ n_oblig[t,i]·Δτₜ // непрерывный генератор P = expm(G · Δt) // матричная экспонента → матрица переходов

Опция monotonic=True применяет к каждой строке монотонную регуляризацию через энтропийный пулинг (M1): вероятность дефолта должна расти с ухудшением рейтинга. Встроено в RatingEngine через from_cohort_data().

M3

Очистка спектра по Марченко—Пастуру — spectrum.py

Шумоподавление

Эмпирическая матрица корреляций при ограниченной истории сильно зашумлена. Закон Марченко—Пастура описывает распределение собственных значений чисто случайной матрицы. Всё, что попадает внутрь этого диапазона — шум, и его собственные значения «схлопываются» в одно среднее; остаётся только сигнал.

λ± = σ²(1 ± √(1/q))², q = T/N // границы спектра чистого шума f(λ) = √((λ₊−λ)(λ−λ₋)) / (2πσ²λ/q) // плотность Марченко-Пастура собственные значения < λ₊ → заменить средним (шум)
Реальный результат (демо)Для матрицы 40×40 из чисто случайных данных (T=80) метод оставил 0 сигнальных факторов — что абсолютно верно: в случайных данных нет структуры выше границы шума. На реальных коррелированных данных метод сохраняет ровно столько факторов, сколько отражают истинную структуру.

Два режима: mp_edge (надёжный, по умолчанию) и hist_mse (точное соответствие arpym). Встроено в TransactionGraph.get_correlation_matrix(denoise=True). Зависимость от skrmt заменена на собственную реализацию.

Источник ARPM: estimation.spectrum_shrink · Тест: test_spectrum_shrinkage · ETL-этап: 20

M4

Гибкие вероятности (Flexible Probabilities) — conditional_fp.py

Режимное взвешивание

Раньше история стресса взвешивалась гладким гауссовым ядром — приближение. Условные гибкие вероятности вместо этого точно воспроизводят условное среднее И дисперсию через энтропийный пулинг (M1) — строгое определение ARPM. crisp_fp даёт жёсткий «оконный» вариант.

p_crisp = жёсткое окно массы α вокруг z* // сценарии «как сегодня» m, s² = условные среднее и дисперсия при p_crisp p_cond = argmin KL(p ‖ p_prior) при E[z]=m, E[z²]=m²+s²
Исправленная ошибка устойчивостиПри очень малом α окно могло оказаться пустым → вектор вероятностей становился нулевым и тихо вырождался в априорный. Добавлен запасной механизм «ближайшего сценария»: если в окно ничего не попало, вся масса отдаётся ближайшему к z* сценарию.

Встроено в FlexibleProbsCalibrator(weighting_method="conditional_fp"). Функция effective_scenarios(p) сообщает эффективное число сценариев — индикатор того, насколько «узко» режим обусловил историю.

Источник ARPM: estimation.conditional_fp / crisp_fp · Тест: test_conditional_fp

M5

Факторные нагрузки — low_rank_corr.py

Факторная структура

Раньше нагрузки факторной копулы (betas) задавались вручную (равные = «одинаково важные»). Теперь fit_factor_loadings оценивает матрицу нагрузок (n, k) из целевой матрицы корреляций (например, корреляции графа транзакций) методом низкоранговой + диагональной аппроксимации.

C ≈ β βᵀ + diag(1 − rowSumSq(β)), β формы (n, k) ∑ₖ βᵢₖ² < 1 // положительная идиосинкразическая дисперсия — копула-готова
Реальный результат (демо)Из корреляционной матрицы 20×20, построенной по известным нагрузкам β≈0.5, метод восстановил нагрузки формы (20,1) со средним |β| = 0.467 — близко к истинному значению.
Исправленная числовая вырожденностьНа почти единичной корреляции «сырая» подгонка возвращала строки с ∑β²=1 (нулевая идиосинкразическая дисперсия), что MultiFactorCopula отвергает. Добавлен предел нормы строки max_row_norm=0.999, гарантирующий положительную идиосинкразическую дисперсию.

Источник ARPM: estimation.low_rank_diag_conditional_corr · Тест: test_low_rank_corr · ETL-этап: 25

M6

Меры зависимости — dependence.py

Ранговая зависимость

Мера Швейцера—Вольфа — ранговый показатель зависимости в диапазоне [0,1], измеряет L1-расстояние между эмпирической копулой и независимостью. В отличие от корреляции Спирмена, ловит немонотонную зависимость. copula_invariance_test — тест на сериальную зависимость (для инварианта зависимость на всех лагах должна быть ≈0).

σ = 12/g² · ∑|C(u,v) − uv| ∈ [0,1] // расстояние эмпирической копулы до независимости
Независимость → 0Демо: 1500 независимых точек → σ = 0.040 ✓ (верно, ≈0).
Полная зависимость → 1Демо: комонотонные данные → σ = 0.996 ✓ (верно, ≈1).

Реализация векторизована (O(g² log g)) с ограничением сетки для масштаба и защитой от почти константных столбцов. Подробнее об ошибке нормировки в оригинале — см. раздел «Найденная ошибка в ARPM».

Источник ARPM: statistics.schweizer_wolff (ошибка исправлена) · Тест: test_dependence_measures

Калибровка копулы по данным — copula_calibration.py

Калибровка

Отвечает на вопрос «откуда взялись параметры копулы?». Полный конвейер: наблюдаемые дефолты → эмпирические меры зависимости → параметры семейств копул → рекомендация семейства по качеству подгонки.

Преобразования τ → параметр (точные формулы)

Clayton: θ = 2τ / (1 − τ) // θ(τ=1/3) = 1.000 ✓ Gaussian: ρ = sin(πτ/2) // ρ(τ=0.5) = 0.707 ✓ ρ_D = (p_AB − p_A·p_B) / √(p_A(1−p_A)p_B(1−p_B)) // корреляция дефолтов

Эмпирические меры: Кендалл τ, Спирмен ρ, Швейцер—Вольф, корреляция дефолтов, наблюдаемая vs независимая частота совместного дефолта, нижне-хвостовая со-частота. Сравнение семейств (Gaussian / Student-t / Clayton) с проверкой границ Фреше и рекомендацией.

Восстанавливает реальную зависимостьНезависимые данные → рекомендация ≈нулевой зависимости (ρ≈−0.01, default_corr≈0.004). Данные с общим шоком → рекомендация копулы с тяжёлыми хвостами (Student-t, ρ=0.382), эмпирическая корреляция дефолтов 0.256 (>>0), наблюдаемый совместный дефолт (0.049) корректно превышает независимый (0.019).

Находка

Найденная и исправленная ошибка в опубликованном ARPM

Bug fix

При портировании schweizer_wolff обнаружилось, что оригинальная функция arpym возвращает значения > 1 на независимых данных — нарушая определение меры (σ обязана быть в [0,1]).

Проверка против известных теоретических значений

СлучайИстинное σНаш портОригинал arpymВывод
Независимость0.0000.0591.125arpym неверно
Полная зависимость1.0000.9510.951оба верно
Корень проблемыОригинал суммирует по индексам i,k = 0..j−1 с порогами i/j, что рассогласовывает эмпирическую копулу и копулу независимости на разных сетках. Мера-индикатор зависимости, дающая 1.12 для независимых данных, — сломана. Наш порт использует 1-индексные пороги i ∈ 1..g, что выравнивает обе копулы на одной сетке.
ИтогНаш порт корректен для обоих случаев с известным ответом. Расхождение задокументировано в docstring модуля как намеренное и обоснованное.
