# Modular ETL Notebooks

One notebook per pipeline stage so **different people can own different parts**
of the credit-risk computation and work independently. Stages communicate only
through a shared on-disk **artifact store** (`output/etl/`), so you can run, re-run,
or debug a single stage without touching the others — as long as that stage's
upstream artifacts already exist.

## Notebooks

| Notebook | Stage | Owner | Reads → Writes |
|---|---|---|---|
| `00_ingest.ipynb` | `stage_00_ingest` | Data Engineer | — → `persons`, `transactions` |
| `10_pd.ipynb` | `stage_10_pd` | ML Engineer | `persons`,`transactions` → `persons_scored` |
| `20_graph.ipynb` | `stage_20_graph` | Data Scientist · graph | `persons_scored` → `corr_matrix` *(arpym #2: MP denoising)* |
| `30_copula.ipynb` | `stage_30_copula` | Data Scientist · copula | `persons_scored`,`corr_matrix` → `joint_default_matrix` |
| `40_transitions.ipynb` | `stage_40_transitions` | Risk Analyst / Credit Risk | `persons_scored` → `transition_matrix`,`ratings` *(arpym #1: generator estimator)* |
| `50_metrics.ipynb` | `stage_50_metrics` | Risk Analyst | `persons_scored`,`corr_matrix` → metrics |
| `99_run_all.ipynb` | all | — | runs the full chain |

## How to use

1. **First time / full run:** open `99_run_all.ipynb` and run all cells. This
   populates `output/etl/` with every artifact.
2. **Work on one stage:** open that stage's notebook. Its upstream artifacts must
   already exist in the store (run `99_run_all.ipynb` once, or the earlier
   stage notebooks, to create them). Edit the corresponding `src/` module, then
   re-run just this notebook to see your change.
3. **Hand-off between teams:** the artifact store *is* the contract. The ML
   Engineer hands the Risk Analyst a `persons_scored` table; the Risk Analyst
   never needs the ML code, only the artifact.

## The two new arpym concepts in the pipeline

- **#2 Marčenko-Pastur spectrum shrinkage** — `20_graph.ipynb`, `denoise=True`.
  Denoises the correlation matrix (random-matrix theory) before it reaches the
  copula. The notebook prints the conditioning improvement vs the raw matrix.
- **#1 Continuous-time credit transition estimator** — `40_transitions.ipynb`.
  Supply a `migration_events` table (period / from_state / to_state / count) and
  the annual transition matrix is estimated with the rigorous generator +
  entropy-regularised monotonicity method instead of the baseline.

## Regenerating the notebooks

The notebooks are generated by a script so they stay in sync with the stage API:

```bash
python notebooks/_build_notebooks.py
```

## Notes

- Notebooks add the project root to `sys.path` (they live one level below it).
- The artifact store uses parquet when a parquet engine is installed, else CSV —
  no extra dependency required.
